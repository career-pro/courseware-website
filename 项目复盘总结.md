# 都在这了课件站 项目复盘总结

## 项目概述

为小学老师搭建的课件资源展示网站，支持主题班会、课件、绘本三个分类，免费资源直接展示下载链接，付费资源通过兑换码解锁。

- **域名**：course.careerease.cn
- **上线时间**：2026年6月
- **技术栈**：Next.js 16 + TypeScript + Tailwind CSS 4 + Vercel 部署

---

## 一、需求梳理

1. 三个课件分类：主题班会、课件、绘本
2. PPT 以图片形式展示，隐藏百度网盘下载链接
3. 免费资源：直接展示下载链接；付费资源：兑换码解锁
4. 兑换码规则：A 码可解锁 1 个课件，B 码可解锁 2 个课件，次数用完失效
5. 搜索 + 标签筛选
6. 管理后台：课件 CRUD、兑换码生成
7. 导航栏：Logo + 首页 / 主题班会 / 课件 / 绘本 + 联系我们（微信号）
8. 所有上传图片自动加水印 "@都在这了"（斜对角、居中、半透明）
9. 设计风格：明亮、多彩、适合小学老师群体

## 四、核心功能实现

### 4.1 数据模型

```typescript
// 课件
interface Course {
  id: string;
  title: string;
  category: 'meetings' | 'courseware' | 'picture-books';
  coverImage: string;       // 封面图 URL
  detailImages: string[];   // 详情图列表
  detailInfo: string;       // 详细信息
  downloadLink: string;     // 下载链接（百度网盘链接+提取码）
  tags: string[];           // 标签
  isFree: boolean;          // 是否免费
  createdAt: string;
}

// 兑换码
interface RedeemCode {
  code: string;             // 兑换码，如 A123456
  type: 'A' | 'B';          // A=1次, B=2次
  remainingUses: number;    // 剩余可用次数
  usedFiles: string[];      // 已解锁的课件 ID 列表
  createdAt: string;
}
```

### 4.2 存储架构

```
本地开发（无环境变量）→ 内存变量模拟
生产环境（有 KV/Redis 环境变量）→ Vercel KV → Upstash Redis
```

通过 `src/lib/kv.ts` 统一抽象，根据环境变量自动切换存储后端。图片存储同理，有 `BLOB_READ_WRITE_TOKEN` 则用 Vercel Blob，否则存本地 `public/` 目录。

### 4.3 图片水印

使用 sharp 库在服务端给上传的图片添加水印。流程：

1. 解析上传的图片，获取尺寸
2. 用精简体中文字体渲染 "@都在这了" 文字图片
3. 将文字居中放置、旋转 -25 度
4. 将水印层以 12% 透明度合成到原图上
5. 输出 PNG 上传到 Vercel Blob

### 4.4 兑换码系统

- 生成：管理后台选择 A 码（1 次）或 B 码（2 次），随机生成字母+数字组合
- 验证：POST `/api/unlock`，检查码是否存在、次数是否用完、是否已解锁该课件
- 解锁后状态存到浏览器 localStorage，全站共享

### 4.5 管理后台

通过密码保护（ADMIN_PASSWORD 环境变量），支持：
- 课件管理：添加/编辑/删除课件，上传封面图和详情图
- 兑换码管理：生成单个/批量兑换码，查看使用状态

## 五、遇到的问题和解决方案

### 问题 1：Vercel 部署 TypeScript 类型报错

**现象**：本地开发正常，Vercel 构建时报 TypeScript 类型错误。

**原因**：本地 IDE 的类型检查和 Vercel 构建时的 `tsc` 检查严格程度不同。

**解决**：每次提交前确认类型正确，不依赖 `any` 类型。

### 问题 2：上传接口 500 错误

**现象**：生产环境上传图片返回 500。

**原因**：加了 Vercel Blob Storage 后，环境变量 `BLOB_READ_WRITE_TOKEN` 没有自动注入。代码检测到没有 Token，走了本地文件存储路径，但 Vercel 文件系统是只读的，报 `EROFS: read-only file system`。

**解决**：在 Vercel 项目的 Settings → Environment Variables 中确认 `BLOB_READ_WRITE_TOKEN` 存在，并增加 Vercel 环境检测逻辑，在缺少配置时返回明确错误提示。

### 问题 3：水印不显示

**现象**：上传成功后图片没有水印。

**原因**：最开始用 SVG 的方式在图片上绘制文字水印，SVG 中的 `@都在这了` 中文字符在 Vercel 服务器上没有对应的系统字体，渲染出来是空白。

**解决过程**：
1. 尝试在 SVG 中用 base64 嵌入字体 → librsvg 不支持 `@font-face` 的 `data:` URI
2. 尝试用文件路径引用字体 → 同样不支持
3. 最终方案：放弃 SVG，改用 sharp 的原生文字渲染 API (`sharp({ text: {...} })`)，加载精简体字 OTF 文件直接渲染文字

**精简体字**：用 Python fonttools 从 17MB 完整字体中提取 "@都在这了" 4 个字符，得到 4.8KB 的迷你字体文件，不会增加部署包体积。

### 问题 4：水印合成报尺寸错误

**现象**：`Image to composite must have same dimensions or smaller`

**原因**：水印文字放在一个与原图相同尺寸的透明画布上，旋转 -25 度后 sharp 自动扩展画布以容纳旋转后的内容，导致水印层比原图大，无法合成。

**解决**：改为将文字放在一个刚好容下它的小画布上旋转，旋转后再放到原图尺寸的透明画布上居中对齐，最后合成。

### 问题 5：sharp 版本兼容

**现象**：不同版本的 sharp 在 TypeScript 类型定义和 Vercel 运行时上有兼容性差异。

**解决**：最终锁定 sharp 0.33.5，对缺失的类型定义加 `@ts-expect-error` 注释。

### 问题 6：npm 安装权限问题（Windows）

**现象**：`npm install` 时报 `EPERM: operation not permitted`，无法删除 `node_modules/sharp`。

**原因**：Windows 系统下 sharp 的二进制文件被进程锁定（可能是杀毒软件或编辑器）。

**解决**：手动 `rm -rf node_modules` 后重新安装。

### 问题 7：兑换码解锁状态跨页面丢失

**现象**：在分类页卡片上解锁了课件，点进详情页还需要再次解锁。

**原因**：两个页面各自维护独立的 React state，页面跳转后状态丢失。

**解决**：创建 `useUnlocked` Hook，将解锁的课件 ID 集合存到 `localStorage`，所有页面共享同一份数据。

### 问题 8：首页设计多次返工

**现象**：首页文字设计被多次否定（渐变色、不同字号、不同字重等）。

**解决**：最终确定极简方案——两行相同字号/颜色的文字 + 一条细分隔线。

## 八、经验总结

1. **Vercel 部署的注意事项**：环境变量不会因为加了 Storage 就自动注入，需要手动确认。文件系统只读，不能依赖本地文件存储方案。

2. **服务端图片处理的坑**：依赖系统字体的方案（SVG 文字）在 serverless 环境不可靠，尽量用原生 API 加载打包的字体文件。

3. **TypeScript 紧约束**：Vercel 构建会严格检查类型，`any`、隐式类型、多余属性都会报错阻断部署。

4. **状态持久化**：跨页面的用户状态需要 `localStorage` 或后端存储，不能只靠 React state。

5. **快速迭代策略**：先让功能跑通，再优化体验和设计。不要一开始就追求完美。
